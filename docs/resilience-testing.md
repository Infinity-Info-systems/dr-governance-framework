# Resilience Testing

## Purpose

Resilience testing proves that recovery plans work and exposes gaps before a real event does.

## Testing Types

- tabletop
- component recovery
- full failover
- failback
- dependency validation
- rehearsal of decision points

## Questions To Ask

- how often is recovery tested?
- what failed in prior tests?
- which dependencies were missed?
- what was improved afterward?

## Evidence To Collect

- test plan
- test results
- remediation actions
- repeat-test evidence

## Testing Outcome

Resilience testing should increase confidence and expose gaps before a real event does.
